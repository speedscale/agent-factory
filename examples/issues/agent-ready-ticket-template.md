# Agent-Ready Ticket Template

Use this structure for issues intended for autonomous processing.

## Metadata

- Repo: `<owner>/<repo>`
- Issue URL: `<link>`
- Issue ID: `<id>`
- Labels: `agent-ready`, `bug`, `validation-required`

## Problem Statement

What is broken and why it matters.

## Observed Behavior

Specific current behavior with examples.

## Expected Behavior

What should happen instead.

## Impact

User/system impact and urgency.

- [ ] check 1
- [ ] check 2
- [ ] check 3

## Validation Notes

Any proxymock dataset/traffic details required.
